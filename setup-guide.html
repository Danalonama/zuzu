לחבר את הטופס למערכת אמיתית

בסוף התהליך: אנשים יוכלו להוסיף אירועים (וגם לגרור צילום מסך מוואטסאפ), ההגשות יגיעו לגיליון שלך, ואת מאשרת בלחיצה אחת. הכל חינמי, בלי שרת. ⏱️ ~10 דקות.

0 מתוך 6 הושלמו
01

יוצרים גיליון Google חדש

פה יישבו כל האירועים שמוגשים
  1. נכנסים ל-sheets.new (עם חשבון Google שיהיה הבעלים).
  2. נותנים שם לקובץ, למשל Zuzu events.
  3. למטה־שמאל, לוחצים דאבל-קליק על שם הלשונית ומשנים ל-events (אנגלית, אותיות קטנות).
  4. בשורה הראשונה (שורה 1) מדביקים את הכותרות הבאות, כל אחת בתא נפרד מ-A עד L:

    timestamp · approved · date · time · title · host · category · venue · city · region · url · price

  5. מסמנים את עמודה B (approved) → תפריט Insert ▸ Checkbox. עכשיו לכל אירוע חדש יהיה תיבת סימון שאת מסמנת כדי לפרסם.
02

מדביקים את קוד השרת

הקובץ zuzu-events.gs מתיקיית backend
  1. באותו גיליון: תפריט Extensions ▸ Apps Script. נפתח עורך קוד בלשונית חדשה.
  2. מוחקים את כל מה שכתוב ב-Code.gs.
  3. פותחים את הקובץ backend/zuzu-events.gs (מהפרויקט), מעתיקים את כל התוכן, ומדביקים בעורך.
  4. לוחצים על אייקון השמירה 💾.
אם אין לך את הקובץ פתוח — תגידי לי ואני אציג לך אותו כאן להעתקה.
03

מוסיפים את מפתח ה-AI לסריקת צילומים

רק זה מפעיל את "גררו צילום מסך"
  1. משיגים מפתח ב-console.anthropic.com → Settings ▸ API Keys ▸ Create Key → מעתיקים (מתחיל ב-sk-ant-).
  2. בעורך Apps Script: Project Settings (גלגל שיניים ⚙ בצד) → גוללים ל-Script Properties → Add script property:
Property
ANTHROPIC_API_KEY
Value
sk-ant-…

לוחצים Save script properties.

חשוב: המפתח נשמר רק כאן, בצד השרת שלך — הוא לא נחשף באתר ולא לאף גולש. אל תשלחי אותו לאף אחד (גם לא אליי).
רוצה לחסוך? אפשר לדלג על השלב הזה — הכל יעבוד חוץ מסריקת הצילומים (אפשר תמיד למלא ידנית). להוזלה: בקוד אפשר להחליף sonnet ב-haiku.
04

מפרסמים כ-Web App

הופך את הקוד לכתובת אינטרנט
  1. למעלה-ימין: Deploy ▸ New deployment.
  2. ליד "Select type" לוחצים על גלגל השיניים ⚙ → בוחרים Web app.
  3. ממלאים:
    Execute as
    Me
    Who has access
    Anyone
  4. לוחצים Deploy ומאשרים את בקשת ההרשאות (זה הקוד שלך — בטוח).
  5. מעתיקים את ה-Web app URL שמופיע — נראה כמו https://script.google.com/macros/s/AKfy…/exec
שמרי את הכתובת הזו — נצטרך אותה בשלב הבא.
05

מחברים את הכתובת לאתר

שורה אחת ב-index.html

הכי קל: שלחי לי את הכתובת בצ׳אט ואני אחבר תוך שנייה. או לבד:

  1. פותחים את index.html.
  2. מחפשים (Ctrl/⌘+F) את השורה: const ZUZU_API = "";
  3. מדביקים את הכתובת בין הגרשיים:

    const ZUZU_API = "https://script.google.com/macros/s/AKfy…/exec";

  4. שומרים ומעלים מחדש את האתר.
06

בודקים שהכל עובד

הגשה ניסיונית → אישור
  1. באתר לוחצים "הוסיפו אירוע", גוררים צילום מסך של אירוע מוואטסאפ ולוחצים "נתחו אירועים" (או ממלאים ידנית) ושולחים.
  2. נכנסים לגיליון — אמורה להופיע שורה חדשה עם תיבת approved לא מסומנת.
  3. מסמנים ✓ את approved.
  4. מרעננים את האתר — האירוע מופיע בלוח. 🎉
זה נשאר ככה תמיד: כל הגשה נכנסת לגיליון לא-מאושרת → את מסמנת ✓ → היא עולה לאתר. להסרה: מבטלים את הסימון או מוחקים את השורה.
